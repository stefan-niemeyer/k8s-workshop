ConfigMap

  • ConfigMap für nicht vertrauliche Daten
  • Werden als Key-Value-Paar abgelegt
  • Pods können Inhalt als Umgebungsvariablen oder Volume verwenden

Doku auf kubernetes.io und hier

binaryData für Binärdaten. ConfigMap max. 1MiB groß

ConfigMap (property-like)


apiVersion: v1
kind: ConfigMap
metadata:
  name: db-config
  labels:
    app: configmap-demo
    app.kubernetes.io/name: db-config
data:
  DB_SERVER: "orasrv.example.com"
  # Value muss ein String sein, daher mit Anfuehrungszeichen
  DB_PORT: "1234"
  DB_URL: "jdbc:oracle:thin:@orasrv.example.com:1234/MY_DB"

ConfigMap (file-like)


apiVersion: v1
kind: ConfigMap
metadata:
  name: game-demo
  labels:
    app: configmap-demo
    app.kubernetes.io/name: game-demo
data:
  # property-like keys
  player_initial_lives: "3"
  ui_properties_file_name: "user-interface.properties"

  # file-like keys
  game.properties: |
    enemy.types=aliens,monsters
    player.maximum-lives=5

  user-interface.properties: |
    color.good=purple
    color.bad=yellow
    allow.textmode=true

Pod mit ConfigMap


apiVersion: v1
kind: Pod
metadata:
  name: configmap-demo-pod
  labels:
    app: configmap-demo
    app.kubernetes.io/name: configmap-demo-pod
spec:
  containers:
    - name: demo
      image: alpine
      command: ["sleep", "3600"]
      envFrom:
        - configMapRef:
            name: db-config
      env:
        - name: DIREKTER_WERT
          value: "Hallo Leute"
        - name: PLAYER_INITIAL_LIVES
          # Beachte: Var-Name != Key in ConfigMap
          valueFrom:
            configMapKeyRef:
              name: game-demo
              key: player_initial_lives
        - name: UI_PROPERTIES_FILE_NAME
          valueFrom:
            configMapKeyRef:
              name: game-demo
              key: ui_properties_file_name
      volumeMounts:
        - name: config
          mountPath: "/meine-config-dateien"
          readOnly: true
        - name: config-einzeln
          mountPath: "/etc/game.properties" # Dateiname
          subPath: "game.properties"        # Key in ConfigMap
          readOnly: true
  volumes:
    # Volumes werden auf Pod Ebene definiert und dann im Container gemountet
    - name: config
      configMap:
        name: game-demo
        # Liste mit Keys aus der ConfigMap die als Datei gemountet werden sollen
        items:
          - key: "game.properties"
            path: "meine-game.properties"
          - key: "user-interface.properties"
            path: "meine-user-interface.properties"
    - name: config-einzeln
      configMap:
        name: game-demo

ConfigMap benutzen


            # ConfigMaps und Pod anlegen
            $ k apply -f cm/db-demo-cm.yaml -f cm/game-demo-cm.yaml
            $ k apply -f cm/configure-pod.yaml
        

            # Shell im Pod öffnen
            $ k exec -it configmap-demo-pod -- sh
