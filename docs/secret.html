Secret

  • Secrets für vertrauliche Daten
    (sind NICHT verschlüsselt)
  • Format und Verwendung ähnlich ConfigMap

Doku auf kubernetes.io

stringData für Klartext, sonst base64 encoded. Secret max. 1MiB groß. Es gibt verschiedene Arten von Secrets. Je nach Typ werden bestimmte Keys erwartet, bspw. `username` und `password` bei `kubernetes.io/basic-auth` `ssh-privatekey` bei `kubernetes.io/ssh-auth` `.dockercfg` bei `kubernetes.io/dockercfg`

Secret (Opaque)


apiVersion: v1
kind: Secret
metadata:
  name: age-secret
  labels:
    app: secret-demo
    app.kubernetes.io/name: age-secret
type: Opaque
stringData:
  age-key: "AGE-SECRET-KEY-1234567890ABCDEFGHIJKLMNOPQRSTUVWXYZ"

Secret (dockercfg)


apiVersion: v1
kind: Secret
metadata:
  name: dockercfg-secret
  labels:
    app: secret-demo
    app.kubernetes.io/name: dockercfg-secret
type: kubernetes.io/dockercfg
data:
  .dockercfg: eyJhdXRocyI6eyJodHRwczovL2V4YW1wbGUvdjEvIjp7ImF1dGgiOiJvcGVuc2VzYW1lIn19fQo=

Pod mit Secret


apiVersion: v1
kind: Pod
metadata:
  name: secret-demo-pod
  labels:
    app: secret-demo
    app.kubernetes.io/name: secret-demo-pod
spec:
  containers:
    - name: demo
      image: alpine
      command: ["sleep", "3600"]
      env:
        - name: AGE_KEY
          valueFrom:
            secretKeyRef:
              name: age-secret
              key: age-key
#      Geht auch mit envFrom, wie bei ConfigMaps
#      envFrom:
#        - secretRef:
#            name: mein-secret
      volumeMounts:
        - name: config
          mountPath: "/meine-secrets"
          readOnly: true
        - name: config-einzeln
          mountPath: "/etc/.neues-dockercfg"
          subPath: ".dockercfg"
          readOnly: true
  volumes:
    - name: config
      secret:
        secretName: dockercfg-secret
    - name: config-einzeln
      secret:
        secretName: dockercfg-secret

Secret benutzen


            # Secrets und Pod anlegen
            $ k apply -f secret/age-secret.yaml -f secret/dockercfg-secret.yaml
            $ k apply -f secret/secret-pod.yaml
        

            # Shell im Pod öffnen
            $ k exec -it secret-demo-pod -- sh
